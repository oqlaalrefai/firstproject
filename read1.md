#Markdown language
Markdown is a way to style text on the web

Markdown is just regular text with a few non-alphabetic characters thrown in, like # or *

You can use Markdown most places around GitHub:
1- Gists
	easy method to share snippets or excerpts of data with others
2- Comments in Issues and Pull Requests
3- Files with the .md or .markdown extension

ex :
1-text:
# tom make header text (level one header you have 6 level)
**bold text**
__bold text__ 
*italic text*
_italic text_
to make a link [link to Google!](http://google.com)


2-to make ordered lists:
in three way:
1-using star(*)
2-using numbers (1- ,2- ,...)
3-using dash (- )

And if you have sub points, put two spaces before the dash or star:
output:
 o Like this
 o And this


3-images
![for end user]
(image link)

4-qoutes

If you'd like to quote someone, use the > character before the line


5- code
meltiple line of code:
A-
```
source of code
```
B-You can also simply indent your code by four spaces:

C-Inline code
text `code` text

6-Task Lists
- [x] this is a complete item
- [ ] this is an incomplete item

7-Tables
You can create tables by assembling a list of words and dividing them with hyphens - (for the first row), and then separating each column with a pipe |

8-Username @mentions

9-Automatic linking for URLs

10-Strikethrough
Any word wrapped with two tildes (like ~~this~~) will appear crossed out.
